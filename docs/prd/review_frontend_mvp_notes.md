# 复习前端 MVP 备注

这份文档只整理当前复习系统对应的前端想法，不讨论视觉设计，也不讨论完整产品。

目标很简单：

- 让现有 `review_state -> review_bundle_builder -> review_state_manager` 这条链路能被一个最小前端直接接起来
- 不再反复改底层结构

---

## 1. 前端当前应该接什么

现在前端最适合直接读取：

- `review_bundle_builder.py` 输出的 `review_bundles`

原因：

- 它已经不是底层状态
- 也不是纯调度分数
- 而是“当前这一轮该给学生看的复习单元”

前端不用自己再拼：

- 知识点卡片
- 题目
- 隐藏答案
- 按钮动作

这些都已经在 bundle 里整理好了。

---

## 2. 两种入口

### 2.1 知识点优先 `leaf_first`

页面顺序：

1. 显示叶子卡片
2. 显示该知识点下的 2~3 道题
3. 每道题默认隐藏答案
4. 学生点击按钮后，再请求下一轮 bundle

单个 bundle 最少包含：

- `leaf_card`
- `questions[]`
- `student_actions`

知识点按钮：

- `掌握很熟练`
- `还要练题`
- `暂时跳过`

题目按钮：

- `显示答案`
- `这题做对了`
- `这题没做对`
- `部分会做`
- `先跳过`

### 2.2 题目优先 `question_first`

页面顺序：

1. 先显示题目
2. 题目答案默认隐藏
3. 下面显示 1~2 个相关知识点卡片
4. 学生做题后点反馈按钮

适合学生只想先刷题。

---

## 3. 最小页面结构

一个最小页面其实只需要三块：

### 3.1 顶部控制区

- 当前模式切换
  - `知识点优先`
  - `题目优先`
- 当前 bundle 序号
- 一个“刷新下一轮”按钮

### 3.2 主展示区

如果是 `leaf_first`：

- 一张知识点卡片
- 下面一组题目卡片

如果是 `question_first`：

- 一张题目卡片
- 下面一组关联知识点卡片

### 3.3 反馈区

所有按钮都不需要前端自己定义语义，直接读 bundle 里的：

- `action`
- `target_type`
- `target_id`
- `result`

前端点击后只要把这些参数回传给后端即可。

---

## 4. 交互原则

### 4.1 答案默认隐藏

这是现在系统的固定前提。

前端应该先显示：

- 题目
- 学生自己的答案（如果有）

答案和解析放在可展开区：

- 默认 `revealed = false`
- 点击 `显示答案` 后展开

### 4.2 先反馈，再刷新

推荐交互：

1. 学生点按钮
2. 前端调用 `review_state_manager`
3. 后端返回新状态
4. 前端再调用 `review_bundle_builder`
5. 刷新当前轮推荐

这样前端永远不用自己算优先级。

### 4.3 本轮即时反馈必须明显

这次已经在底层实现：

- `还要练题` 会让当前知识点和关联题在本轮继续靠前
- `wrong` 会让当前题和绑定知识点立即强化
- `掌握很熟练` 会让当前知识点本轮后移

所以前端要做的不是“解释算法”，而是让学生能明显看见：

- 这一点刚点完，下一屏真的变了

---

## 5. 前端暂时不用做的事

先不要做：

- 完整学习历史页
- 数据分析图表
- 家长端汇总
- 多学生切换
- 拖拽式知识树编辑

这些都不是 MVP。

---

## 6. 当前最小后端接口抽象

如果后面要封装接口，最小只需要这两类：

### 6.1 获取复习包

输入：

- `review_state`
- `mode`

输出：

- `review_bundles`

### 6.2 提交反馈

输入：

- `action`
- `target_type`
- `target_id`
- 可选 `result`

输出：

- `updated_payload`

前端拿到 `updated_payload` 之后，再重新请求 bundle。

---

## 7. 当前建议

当前阶段不要再大改底层结构。

更合理的顺序是：

1. 先固定现在这套 bundle / action 结构
2. 前端直接按这个结构接一个最小页面
3. 真跑一轮之后，再决定要不要继续细化字段

一句话：

现在前端最重要的不是“设计多漂亮”，而是尽快验证：

- 学生看这种复习单元顺不顺
- 点按钮后的反馈够不够直观
- 知识点和题目混合复习是否真的比单刷题更好用
